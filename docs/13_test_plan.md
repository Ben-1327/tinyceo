# 13 Test Plan（テスト計画）

## 目的
- “作業を邪魔しない” の検証
- バランスが理不尽でないか（詰み/単調/運ゲー）
- 実作業連携が気持ちよく作用するか

## 代表プレイテストシナリオ
1. 1日2サイクルしか触らない（忙しい日）
2. 1日4〜6サイクル触る（普段）
3. YouTube多め（Focus/Research切替の価値確認）
4. 受託だけで回す（TechDebt地獄が“遅れて効く”か）
5. SaaSだけで回す（キャッシュ不足→死なないか）
6. 採用連打（固定費過多→学習できるか）

## バランス観測指標
- 倒産率（初週で何%が倒産するか）
- TechDebtが上がった後、制度で戻せるか
- TeamHealthの回復速度
- カードのカテゴリ偏り（単調性）
