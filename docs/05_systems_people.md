# 05 Systems: People（人材・採用・育成）

## 役割（Role）
- Founder（固定）
- Engineer / Designer / Sales / Ops / PM / Support

## ステータス（最小構成）
- Speed（速度）: WorkUnits出力に影響
- Quality（品質）: TechDebt生成/顧客満足に影響
- Comms（連携）: TeamHealth、プロジェクト効率に影響
- AI（AI活用力）: AI制度/ツールの効果を増幅

## 特性（Traits）
社員は 1〜2個の特性を持つ（育成とバランスの核）。
例:
- “AI Native” : AI制度の効果 +20%
- “Perfectionist” : Quality↑ Speed↓
- “Mentor” : 育成速度↑
- “Lone Wolf” : Comms↓（火種）

## 採用
- CEOカード（Hiring）で候補者が提示され、3択で採否/条件提示を決める
- 採用は固定費を増やすが、WorkUnits出力が増える
- 採用の失敗（カルチャーミスマッチ）は TeamHealth を下げる

## 育成
- 自然成長（経験で微増）
- CEOカードで “育成イベント” を発生させる（研修/メンタリング）
- 制度（Onboarding, 1on1, Review）で成長曲線を変えられる

## 離職
- TeamHealthが低いと離職カードが出る
- 対策は「給与」「制度」「負荷調整」
