# 01 Game Design Document (GDD)

## タイトル（仮）
TinyCEO（ティニー・シーイーオー）

## ハイコンセプト
「あなたの実作業が会社の成長に変換される、常駐型ITベンチャー経営シム」

## ターゲット
- PC作業が多い人（エンジニア/PM/デザイナー/オペレーション）
- “経営シム”が好きだが、ガッツリ張り付く時間はない人
- 2時間に1回くらいなら触れる人

## コアファンタジー
- 自分が社長として意思決定し、会社が強くなる
- 採用・制度・AI活用の仕組み化で「放置できる会社」に育つ
- 受託で食いながら、SaaSで未来を作る

## コアメカニクス
- リアル時間で進む背景シミュレーション（軽量）
- 2時間ごとの CEOカード（意思決定）
- 実作業シグナル → Founder/Company の“実行力”に変換
- 経営指標（Cash/Runway, Reputation, TeamHealth, TechDebt, Execution）
- プロジェクト（受託・プロダクト）進行

## 成功の定義（ゲームとして）
- 破産せずに、会社をスケールさせる（まずは10人規模）
- “仕組み化”が進むほど手がかからなくなる
- 最終的に Fundraising / IPO / M&A が選べる土台へ

## 失敗の定義
- Cash が 0 になり支払い不能（倒産）
- ※ただし救済のための「リストラ」「借入」「ピボット」カードで復帰できる余地を残す
